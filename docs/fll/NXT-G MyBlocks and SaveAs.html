NXT-G MyBlocks and SaveAs

By Doug Frevert

Another year and another NXT-G class. This year exposed a bug in creating a MyBlock by SaveAs of another MyBlock.

SaveAs: don't use it to create a new MyBlock.

Don't:

  1. Open an existing MyBlock.
  2. File -> SaveAs -> Enter a new name for the MyBlock.
  3. Edit the MyBlock with the new name. Change something obvious.
  4. Close the MyBlock with the new name.
  6. Use the original and new MyBlock in a NXT-G program.
  7. Run the NXT-G program. You may or may not see anything.
  8. Now open either of the MyBlocks and change something else. The something else becomes part of both MyBlocks; the one edited and the one not edited. The "SaveAs" MyBlocks are linked together and that linkage is hidden.

Instead:

  1. Create a NXT-G program that functions.
  2. Save
  3. Create a MyBlock from the NXT-G program. This is the normal process.
  4. Save the MyBlock
  5. Close the MyBlock
  6. Close the NXT-G program without saving.
  7. Re-open the NXT-G program.
  8. Edit and then create second MyBlock. The two MyBlocks will not be linked in mysterious ways.

Teachers often learn while teaching.

Too bad about no one signing up for "advanced" NXT-G programming. Then again, I'm still learning the simple things.
